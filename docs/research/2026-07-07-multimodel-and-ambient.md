## TOPIC A — Multi-model orchestration patterns

### A1. Tiered cascades / routing

**State of the art:** routing has matured from RouteLLM's original preference-data-trained routers into a small academic/OSS ecosystem, but the *practical* best practice for personal-scale systems has converged on simple two-tier classification, not learned routers.

- [RouteLLM (lm-sys)](https://github.com/lm-sys/RouteLLM) remains the reference OSS framework ([original blog](https://www.lmsys.org/blog/2024-07-01-routellm/)); its trained routers (matrix factorization, BERT-based) need preference data you won't have on day one.
- The main successor-style project is [LLMRouter (UIUC u-lab)](https://github.com/ulab-uiuc/LLMRouter), an open library with 16+ routing models in four categories: single-round, multi-round, **agentic**, and **personalized** routers (KNN/SVM/MLP/matrix-factorization/Elo/graph/BERT). 2026 roundups also describe *RouteProfile* (May 2026), which builds routing profiles from heterogeneous interaction histories ([Maxim's Top-5 LLM routers 2026](https://www.getmaxim.ai/articles/top-5-llm-router-solutions-in-2026/), [awesome-ai-model-routing](https://github.com/Not-Diamond/awesome-ai-model-routing)).
- [vLLM Semantic Router](https://vllm-semantic-router.com/) is the notable infra-level OSS: signal-driven routing composing cost/privacy/latency/safety signals into policies — the "privacy signal" concept (route sensitive content only to local models) maps directly onto a local-first assistant.
- Reported best practices from 2026 router comparisons ([Maxim](https://www.getmaxim.ai/articles/top-5-llm-router-solutions-in-2026/), [Inworld](https://inworld.ai/resources/best-llm-router-ai-gateway)): two-tier classification systems reach >92% routing accuracy; the two failure modes are over-routing (complex task → cheap model) and under-routing (simple task → expensive model); production routers need failover on rate limits, and per-request logging of cost, latency, model choice, and classification confidence.
- Granularity is extending in both directions: a 2026 survey ([arXiv 2606.20295](https://arxiv.org/html/2606.20295v1)) notes cascading has moved from request-level to token-level and into agent workflows, and makes the interesting claim that **cascades can be more robust than a single strong-model call** because the escalation step functions as iterative debugging.

**Pattern for local setups:** small model as always-on classifier/filter (intent, complexity, sensitivity), big model invoked on demand; escalate on low confidence or explicit signals. Learned routers are a later optimization once interaction logs exist to train on.

### A2. Sleep-time compute / offline consolidation

The most directly applicable pattern for the "shell out to Claude Code while idle" architecture; the evidence base is now decent.

- **Letta's core result** ([blog](https://www.letta.com/blog/sleep-time-compute/), [paper materials](https://github.com/letta-ai/sleep-time-compute), [PromptHub summary](https://www.prompthub.us/blog/sleep-time-compute)): an idle-time agent transforms "raw context" into "learned context" so the runtime agent doesn't re-reason from scratch. Reported: Pareto improvement on math benchmarks, accuracy gains up to ~18% on some reasoning tasks, ~2.5× reduction in cost per query. Crucially, the reference architecture is **deliberately heterogeneous: a fast cheap model for the live agent, a stronger slower model for the sleep agent** — exactly the local-Qwen + subscription-Claude split.
- **Artifacts that work**, per Letta's [sleep-time agents docs](https://docs.letta.com/guides/agents/architectures/sleeptime/) and their community [best-practices guide](https://forum.letta.com/t/sleeptime-agents-for-memory-consolidation-best-practices-guide/154):
  - consolidated memory blocks (insert/replace/rethink operations on shared memory);
  - hierarchically tagged entries (`project:x`, `type:decision`);
  - an **archival directory index** (catalog of projects/topics so retrieval is cheap);
  - a **metacognition block** tracking blind spots and retrieval gaps;
  - a "subconscious channel" of passively surfaced related memories.
- **Cadence that works** (same guide): dedup every run; light consolidation at session end; full reorganization weekly; hierarchical rollups monthly. Division of labor: live agent makes quick tactical writes ("user prefers dark mode"), sleep agent does deep consolidation. **Known failure mode: over-consolidation destroys granularity** — merge too aggressively and retrievable detail is lost. Expiry should weight reference count over age. They report Haiku-class models suffice for routine consolidation; the big model is only needed for pattern-finding.
- **Applied to Claude Code:** Letta ships [claude-subconscious](https://github.com/letta-ai/claude-subconscious) ([Cameron's writeup](https://cameron.stream/blog/claude-subconscious/), [Medium overview](https://medium.com/@reliabledataengineering/claude-subconscious-gives-claude-code-a-persistent-memory-that-actually-works-bfd6ff5444e3)) — a background agent that ingests each session transcript asynchronously, updates persistent memory, and pre-stages context injections for the *next* prompt. Claude Code itself now has "AutoDream," an idle-period memory consolidation pass that prunes/merges/refreshes memory files ([MindStudio explainer](https://www.mindstudio.ai/blog/what-is-claude-code-autodream-memory-consolidation)). The pattern is spreading — NousResearch's hermes-agent has an open feature request for a "subconscious observer" explicitly inspired by it ([issue #553](https://github.com/NousResearch/hermes-agent/issues/553)).

### A3. Distillation-into-prompt ("system prompt learning")

- Origin: Karpathy's May 2025 proposal that a third learning paradigm is missing — knowledge in pretraining, habits in finetuning, but human-like "when I hit this kind of problem, try this approach" learning belongs in an **editable system prompt** ([tweet](https://x.com/karpathy/status/1921368644069765486)).
- **Concrete implementation with numbers:** the [optillm SPL plugin](https://github.com/codelion/optillm/tree/main/optillm/plugins/spl) ([announcement](https://huggingface.co/posts/codelion/495839598023200)) has the LLM build and refine a strategy database injected into its own prompt. Results: Arena Hard 29% → 37.6%, AIME24 23.3% → 30%, OptILLMBench 61% → 65%; after 500 queries it had accumulated 129 strategies and refined 97. Works against any OpenAI-compatible endpoint (i.e., against a local MLX/Ollama server too). Caveat: needs hundreds of queries before the strategy DB pays off.
- Related academic work: Stanford/SambaNova's **ACE — "Agentic Context Engineering"** (arXiv 2510.04618) reports ~+10% on agent benchmarks by having a reflector/curator rewrite an evolving "playbook" context, with the key finding that **incremental delta updates beat full prompt rewrites**, which cause "context collapse" (each rewrite compresses away detail) *[background knowledge, not re-verified — the verification search hit the tool outage]*. This converges with Letta's over-consolidation warning from an independent direction: **append/patch, don't rewrite wholesale.**
- Community practice: the OpenClaw ecosystem effectively runs on this pattern — the agent's "operating system" is five hand- and agent-editable markdown files, and the top-cited guide recommends editing them as the primary tuning mechanism ([Lenny's Newsletter complete guide](https://www.lennysnewsletter.com/p/openclaw-the-complete-guide-to-building)).

**Verdict:** the pattern is real, has one measured implementation (SPL) and one paper (ACE), and is de-facto standard practice in the personal-agent community — but nobody has published results specifically for "big cloud model rewrites a small local model's prompt"; that exact asymmetric configuration is still to be validated.

### A4. Draft-verify / critic patterns between small and big models

- **Token level:** Google Research's [speculative cascades](https://research.google/blog/speculative-cascades-a-hybrid-approach-for-smarter-faster-llm-inference/) formalize the hybrid: small model drafts, large model verifies in parallel. Inference infrastructure, not directly buildable across a local/CLI boundary, but it legitimizes the direction.
- **Workflow level:** practitioner guides on local agent stacks consistently split roles — 3–4B models for tool-call parsing, formatting, and selection; big models for multi-step reasoning ([SitePoint local-agents stack guide](https://www.sitepoint.com/the-complete-stack-for-local-autonomous-agents--from-ggml-to-orchestration/)); the 2026 inference-optimization survey documents cascades-with-verification as an emerging agent-workflow pattern ([arXiv 2606.20295](https://arxiv.org/html/2606.20295v1)).
- **For memory writes specifically**, the evidence is indirect but pointed: Letta separates fast tactical writes from a slower consolidating critic (A2); a first-person OpenClaw report found the worst failure was **memory that existed in conversation context but was never durably committed** ([Mario Hayashi](https://blog.mariohayashi.com/p/i-set-up-an-ai-personal-assistant)); the most careful screenpipe review warns automations "write slightly wrong updates with impressive confidence" ([limitations-first review](https://medium.com/@trelvek/screenpipe-review-2026-a-limitations-first-look-at-always-on-desktop-memory-f21d2497d88c)). Together these argue for a two-stage write path: local model drafts memory writes cheaply and immediately; a stronger model audits the accumulated writes in batch. Nobody has published this exact "big-model-as-nightly-auditor-of-small-model-memory-writes" loop with metrics — it's a synthesis, but every component has evidence behind it.

### A5. Headless Claude Code / Codex CLI as a personal batch backend

**What people build:** `claude -p` (`--print`) turns Claude Code into one-shot batch invocations suitable for cron; combined with `--allowedTools` you get a scoped autonomous agent ([MindStudio headless explainer](https://www.mindstudio.ai/blog/claude-code-headless-mode-autonomous-agents), [wmedia guide](https://wmedia.es/en/tips/claude-code-headless-mode-autonomous-agent), [Konishi's CI/CD writeup](https://hidekazu-konishi.com/entry/claude_code_cicd_and_headless_automation.html)). Canonical cron shape: `0 9 * * 1-5 cd /repo && claude -p "check X and summarize" >> log` ([claudefa.st scheduled-tasks guide](https://claudefa.st/blog/guide/development/scheduled-tasks)). Anthropic has productized the pattern as scheduled tasks/routines ([MindStudio on routines](https://www.mindstudio.ai/blog/claude-code-routines-scheduled-tasks-business-automation)); OpenAI equivalently ships **Automations** in the Codex app — cron-syntax schedules, dedicated background worktree, results in a Triage view ([official docs](https://developers.openai.com/codex/app/automations)).

**What breaks — this changed materially in June 2026:**

- **Billing:** as of June 15, 2026, Anthropic bills non-interactive usage (`claude --print`, GitHub Actions, cron jobs, Agent SDK calls) against a **separate monthly Agent SDK credit**: ~$20 on Pro, ~$100 on Max 5x, ~$200 on Max 20x, metered at standard API rates (Sonnet-class ≈ $3/$15 per Mtok), **no rollover**. When exhausted: either overflow API billing (surprise bills) or **hard failures — silent mid-month workflow breakage if unmonitored**. One author cut costs ~80% in a weekend via: route simple steps to Haiku-class models, prompt caching (~90% off repeated input), lower frequency, consolidate sequential calls into one prompt, cap output length ([detailed writeup](https://medium.com/@fukuda.aritomo/your-claude-code-automations-are-about-to-get-a-bill-6a77cf5338f9)). Interactive sessions stay on normal subscription limits.
- **Codex quota mechanics:** usage counts against a 5-hour rolling window plus an independent weekly quota; the 5-hour reset does not restore the weekly pool ([knightli explainer](https://knightli.com/en/2026/04/15/codex-usage-limits-five-hour-weekly-credits/), [InventiveHQ](https://inventivehq.com/blog/codex-cli-usage-rate-limits)). Community pain: rapid quota-drain reports since May 10, 2026 ([OpenAI forum thread](https://community.openai.com/t/codex-users-report-rapid-quota-drains-since-may-10/1380649)) and a bug class where `/status`, the warning banner, and actual errors disagree about remaining quota. Mitigations: flex-tier processing, hiding reasoning events and cutting verbosity, "reset banking" to unblock a rate-limited `codex exec` pipeline ([cost-control guide](https://codex.danielvaughan.com/2026/06/12/codex-cli-rate-limit-reset-banking-usage-optimisation-cost-control-profiles-token-budgets/)).
- **Reliability/safety incidents from the OpenClaw community:** unintended mass Gmail deletion, calendar corruption from automated changes; standard advice is now **never run an autonomous agent on your active personal machine** with broad write access — isolate, and authenticate via scoped keys rather than reusing consumer account credentials ([Lenny's guide](https://www.lennysnewsletter.com/p/openclaw-the-complete-guide-to-building)).

**Best-practice synthesis:** one consolidated prompt per job rather than many small calls; cheapest adequate model per step; cap outputs; cache stable context; monitor credit and alert loudly on failure; treat quotas as a budget with a designed degradation path.

## TOPIC B — "Smart computer" ambient intelligence

### B1. Ambient capture: what exists and what it's worth

- **The consumer cloud products are gone.** Meta acquired Limitless (formerly Rewind) in December 2025 ([TechCrunch](https://techcrunch.com/2025/12/05/meta-acquires-ai-device-startup-limitless/), [9to5Mac](https://9to5mac.com/2025/12/05/rewind-limitless-meta-acquisition/)); the Rewind Mac app stopped recording December 19, 2025 ([rewind.ai's own timeline](https://rewind.ai/what-happened-to-rewind/)); Pendant sales ended ([mlq.ai](https://mlq.ai/news/meta-acquires-ai-wearables-startup-limitless-ending-sales-of-pendant-device/), [Sacra analysis](https://sacra.com/research/why-meta-bought-limitless/)). Lesson: users who invested in cloud ambient memory got rug-pulled — a strong argument for the local-first stance.
- **screenpipe is the surviving option** and is healthy: YC S26, MIT-licensed, 24/7 local screen+audio capture, now reading text via accessibility APIs with OCR as fallback, active releases through June 2026, integrations with OpenClaw and hermes-agent ([GitHub](https://github.com/screenpipe/screenpipe)).
- **Honest utility assessment** (the [limitations-first review](https://medium.com/@trelvek/screenpipe-review-2026-a-limitations-first-look-at-always-on-desktop-memory-f21d2497d88c) is the best single source): OCR misses content; transcription fails on accents/crosstalk/audio routing; DRM and permission-restricted apps don't capture; CPU/battery/storage growth must be empirically tested; summaries may not "save real time"; automations built on capture exhibit confident small errors. "Local data is still data" — you inherit disk-encryption, redaction, and backup-policy obligations. Clear fit: solo operators who live in calls/dashboards and lose time reconstructing their day. Clear non-fit: people who just want meeting notes, or handle sensitive data without time to configure exclusions.
- **Privacy-lighter subset** (frontmost app + window title + idle state, ActivityWatch-style — [activitywatch.net](https://activitywatch.net/) *[background knowledge; the ActivityWatch+LLM utility search hit the tool outage]*): gives the timeline skeleton ("what was I doing when") at negligible resource/privacy cost, without content capture. No strong published evidence either way on LLM-summarized window-title journals; cheap to try, cheap to discard.

### B2. Proactive assistants: what users keep vs disable

**Kept long-term (recurring across sources):**

- **Meeting-prep briefs delivered ~30 minutes before calls** — a headline kept-behavior in Claire Vo's six OpenClaw workflows ([Lenny's guide](https://www.lennysnewsletter.com/p/openclaw-the-complete-guide-to-building)).
- **Morning briefing / inbox+calendar triage at a fixed time** (see B3).
- **Weekly reviews built from meeting notes/activity**, deadline watching, recurring-logistics automation ([Lenny's guide](https://www.lennysnewsletter.com/p/openclaw-the-complete-guide-to-building), [openclaw.ai showcase](https://openclaw.ai/showcase/)).
- **Newsletter/notification summarization into a digest** — the concrete keeper in the Hayashi first-person report ([blog](https://blog.mariohayashi.com/p/i-set-up-an-ai-personal-assistant)), which also kept quick-question access via Telegram and calendar-event *creation from explicit requests*.
- Structural finding: **multiple narrow single-purpose agents beat one generalist** for reliability ("biggest unlock" — Lenny's guide).

**Disabled or regretted:**

- **Autonomous writes to shared mutable state**: documented OpenClaw failure modes are mass Gmail inbox deletion and calendar corruption from automated changes ([Lenny's guide](https://www.lennysnewsletter.com/p/openclaw-the-complete-guide-to-building)). This is precisely why "propose, don't act" wins for email/calendar/files.
- **Interruptive suggestions**: research on proactive programming support finds the core trade-off is intervention benefit vs. the cost of being wrong or intrusive; misreading the user's situation produces disruptive suggestions ([arXiv 2502.18658](https://arxiv.org/pdf/2502.18658)); industry guidance is blunt — too-frequent or irrelevant interruptions get the feature turned off, "knowing when to stay silent is critical" ([Lyzr on proactive agents](https://www.lyzr.ai/glossaries/proactive-ai-agents/)). Mitigations with evidence: batch by context, group related items, infer priority before interrupting, learn from accept/dismiss feedback ([Courier on notification fatigue](https://courier-com.medium.com/notification-fatigue-is-real-and-getting-worse-e4fc248dc29f)).
- **Auto-filing files:** [llama-fs](https://github.com/iyaja/llama-fs) (self-organizing filesystem, watch-mode auto-rename) remains a 2024 hackathon artifact; no maintained 2026 successor found, and the [HN discussion](https://news.ycombinator.com/item?id=40484512) was skeptical of silent moves. The surviving shape is propose-then-confirm organization, not a daemon that moves files.
- **Silent memory loss** as a trust-killer: context held in conversation but never committed to durable memory (Hayashi) — users experience this as the assistant "forgetting," and it erodes the whole premise.

### B3. Briefings and journaling that stuck

- The 2026 daily-briefing category is established: Google shipped **Gemini Daily Brief** in Workspace ([MindStudio explainer](https://www.mindstudio.ai/blog/google-gemini-daily-brief-ai-morning-digest)); head-to-head roundups agree the briefings that stick **triage with urgency reasoning, surface calendar context, and extract tasks/commitments** — mere summarization is not enough ([alfred's tested comparison](https://get-alfred.ai/blog/best-ai-daily-briefing-tools)).
- Retention mechanics: digests survive when they **attach to an existing habit and reading time** rather than inventing a new ritual; measured payoff for newsletter digests is 45–80 min/day down to under 10 ([Readless](https://www.readless.app/blog/morning-newsletter-digest-automation)).
- Scheduled-agent briefings (Claude routines / OpenClaw heartbeats) are the common DIY implementation ([MindStudio on Claude routines](https://www.mindstudio.ai/blog/claude-routines-schedule-autonomous-ai-workflows)).
- **Automated journaling:** evidence is thin. screenpipe positions day-reconstruction as its core value but the sober review questions whether generated summaries save real time. Reading (inference, not sourced): *briefings* (finite, forward-looking, actionable) demonstrably stick; *journals* (archival, backward-looking, rarely re-read) lack retention evidence — unless the journal is an input to something else, e.g. the weekly review or the sleep-time consolidator.

### B4. Genuinely notable 2025–2026 novelties

1. **OpenClaw** — the breakout open-source personal agent ([GitHub](https://github.com/openclaw/openclaw), [Unite.ai review](https://www.unite.ai/openclaw-review/)): local gateway bridging messengers to agents whose "OS" is editable markdown, cron heartbeats, a hosted-wrapper ecosystem, and enough real incidents to generate mature security norms. It validated demand for the "assistant that does things" category — while demonstrating every failure mode of insufficient guardrails.
2. **Memory-first agent infrastructure**: Letta's [claude-subconscious](https://github.com/letta-ai/claude-subconscious) and Claude Code's AutoDream idle consolidation ([MindStudio](https://www.mindstudio.ai/blog/what-is-claude-code-autodream-memory-consolidation)) — sleep-time compute went from paper (2025) to shipped developer tooling (2026).
3. **Platformized scheduled agents with metered batch budgets**: Anthropic's Agent SDK credit split (June 2026) and OpenAI's Codex Automations with Triage ([docs](https://developers.openai.com/codex/app/automations)) — personal cron-agents are now a first-class, but *quota-governed*, product category.
4. **Ambient-capture consolidation**: Meta absorbing Limitless/Rewind leaving local OSS (screenpipe) as the only serious continuity option.
5. **Routing/self-improvement research maturation**: LLMRouter's personalized/agentic router taxonomy, speculative cascades, SPL implementations, ACE's evolving-playbook results *[background]*.

## What to adopt on the target hardware (Apple Silicon; Qwen3-30B-A3B + Qwen3-4B local; Claude Code/Codex CLI on subscription)

RAM feasibility baseline: 30B-A3B at 4-bit ≈ 17–18 GB + Qwen3-4B ≈ 2.5 GB fits the ~20 GB model budget with headroom for an embedding model — every recommendation below respects that; nothing requires a second large resident model.

1. **Rules-first two-tier routing, learned router later.** Qwen3-4B as the always-on classifier (intent, complexity, sensitivity, memory-worthiness) escalating to 30B on low confidence; log every routing decision with confidence/cost/latency. Don't adopt trained routers until months of logs exist. *Cost: zero incremental RAM; 4B inference is effectively free.*
2. **Nightly sleep-time consolidation via `claude -p` — the single highest-value adoption.** One consolidated nightly job reads the day's activity log + transcripts and maintains Letta's proven artifact set: user-profile block, learned-context blocks, tagged archival index, and a metacognition/blind-spots block, all as git-versioned markdown/SQLite the local models load at runtime. Follow the evidenced cadence (dedup nightly, reorganization weekly, rollups monthly) and the two hard-won rules: **delta updates, never full rewrites** (context collapse), and reference-count-weighted expiry. *Feasibility: one capped-output Sonnet-class run/night plus cheaper runs fits well inside a Max-plan Agent SDK credit, especially with prompt caching over the stable memory files.*
3. **Weekly "briefing-pack rewrite" (SPL-lite).** A weekly Claude Code job distills accumulated successes/failures into the small models' decision rules and the 30B's behavioral briefing — system-prompt-learning, implemented as reviewed, diffable markdown patches. Expect payoff only after weeks of accumulation (SPL needed ~500 queries).
4. **Asymmetric draft-verify for memory and triage.** Local 30B drafts memory writes and triage labels immediately (schema/rules-validated locally, never blocking); the nightly big-model pass audits the queue semantically — catching the documented "confidently slightly wrong" and "never actually persisted" failure modes. High-risk side effects stay behind the existing autonomy policy regardless.
5. **Headless-CLI budget discipline with designed degradation.** Treat the Agent SDK credit as a hard monthly batch budget: consolidated prompts, output caps, prompt caching, cheapest-adequate model per step (~80% savings demonstrated); **loud failure alerts** (exhausted credit means silent hard failures); Codex CLI as the overflow lane for code-shaped batch work (mind the 5-hour + weekly windows and quota-reporting bugs). Every scheduled job needs a local-only fallback (skip consolidation, retain raw logs) so offline or quota-out days degrade gracefully instead of breaking.
6. **Ambient capture: start light, defer screenpipe.** Adopt frontmost-app + window-title + idle tracking plus read-only calendar/mail as the timeline skeleton — near-zero resource and redaction cost. Full 24/7 screenpipe capture is proven tech but carries real CPU/battery/disk and exclusion-configuration burden with unproven summary ROI for a student workload; revisit only if "what was I doing/reading" queries become frequent, and configure exclusions before first run.
7. **Proactive surface limited to the evidence-backed keepers:** (a) fixed-time morning briefing with urgency-triaged inbox + calendar + deadlines (triage-with-reasoning, not summaries); (b) meeting-prep brief ~30 min before events; (c) weekly review. Batch everything at predictable times; never interrupt mid-focus; propose-don't-act on anything touching shared state; track accept/dismiss per briefing section and prune what gets ignored. Explicitly skip: auto-filing daemons, streaming notifications, autonomous inbox/calendar mutation (the two documented catastrophic OpenClaw failures).
8. **Adopt the incident-driven security norms early:** scoped `--allowedTools` on every headless invocation, no reuse of consumer credentials in agent contexts beyond what the CLIs themselves manage, and provenance-logged side effects — the OpenClaw community paid for these lessons; Aitvaras's existing autonomy-policy design already matches where the ecosystem landed.

**Research gaps:** ACE paper details, ActivityWatch+LLM field reports, and journaling-retention data were not verified this session due to a tool outage; recommendations 3 and 6 lean partially on background knowledge and inference, flagged above.
